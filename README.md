🎯 Ключевые особенности
1. 📦 Структуры данных
Структура	Назначение
Job	Содержит идентификатор (ID) и URL для обработки
Result	Хранит информацию о выполненном задании: статус, время выполнения, ошибки
2. 🔄 Worker Pool (Fan-out)
Запускается фиксированное количество воркеров (по умолчанию 5)

Каждый воркер конкурентно обрабатывает задания из общего канала jobs

Используется sync.WaitGroup для синхронизации завершения всех воркеров

3. 🌐 Имитация HTTP-запросов
Случайная задержка от 100 мс до 1100 мс (симуляция сетевых вызовов)

10% вероятность ошибки для реалистичности тестирования

4. 📥 Сбор результатов (Fan-in)
Все результаты собираются через единый канал results

Отдельная горутина закрывает канал после завершения всех воркеров

Главная горутина читает результаты через for range

5. 🔒 Синхронизация и безопасность
Каналы обеспечивают потокобезопасную передачу данных

sync.WaitGroup гарантирует корректное завершение всех воркеров

Отсутствие общих изменяемых структур между горутинами

6. 📊 Агрегация и отчет
Подсчет общей статистики (успешные/ошибочные запросы)

Вычисление среднего времени обработки

Детальный вывод по каждому URL
